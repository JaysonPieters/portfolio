The Randomizer

Because it is quite a short game, I wanted to make it replayable, to a certain degree. I decided to do this by making a whole lot of things different each run. I decided to randomize 3 things: The player spawn, the monster spawn and the exit doors.

Here are some of the key feature.

Player Spawn

The way i made the player spawner is quite simple. I used empty gameobjects to mark spawns for the player, then I put all those gameobjects into an array. On load the game picks a random spawnpoint and puts the player there.

More Info

Monster Spawn

I used the same base code from the player spawner, with some minor adjustments. At first I just copy pasted the code but used different spawn points, which worked fine. However, I quickly realized that the monster had a small chance to spawn really close to the player. This made playing the game quite annoying at times, because you would instantly get killed. So I made it so the monster has to choose a spawn that is at least X distance away.

More Info

Random doors

The door spawning was quite different however. I wanted to have the doors be there at all times so the player has to take a risk going for a door, or keep running from the monster hoping to extend the distance between them. The system worked by picking 2 doors. The challange was that it could pick the same door twice, so I had to make a check that if it would pick the same door, it chose a different one.
